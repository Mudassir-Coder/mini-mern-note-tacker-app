03 · MongoDB Schema

Model: Note

One model. Simple. Notice how the fields map directly to your form fields — that's intentional.

📄 models/Note.js

const mongoose = require('mongoose')

const noteSchema = new mongoose.Schema(
  {
    title: {
      type: String,
      required: [true, 'Title is required'],
      minlength: [3, 'Min 3 characters'],
      trim: true,
    },
    content: {
      type: String,
      required: [true, 'Content is required'],
      trim: true,
    },
    tag: {
      type: String,
      enum: ['work', 'ideas', 'personal', 'study'],
      default: 'personal',
    },
  },
  { timestamps: true } // adds createdAt + updatedAt
)

module.exports = mongoose.model('Note', noteSchema)

📋 Field breakdown

title

String, required, min 3 chars, trimmed

trim removes leading/trailing spaces silently

content

String, required, trimmed

No min — short notes are fine

tag

Enum — only accepts 4 values

MongoDB will reject anything outside the list

timestamps

Auto-added: createdAt + updatedAt

Free fields — use createdAt to sort & display dates

⚡ Key insight

Your Mongoose schema and your Zod schema will define the same rules. Zod validates on the frontend before sending the request. Mongoose validates on the backend before saving to the DB. Double protection.

04 · Zod Schema

Frontend Validation

Write this once, get automatic error messages everywhere.

📄 schemas/noteSchema.js

import { z } from 'zod'

export const noteSchema = z.object({
  title: z
    .string()
    .min(3, 'Title needs at least 3 characters')
    .max(80, 'Title is too long')
    .trim(),

  content: z
    .string()
    .min(1, 'Content cannot be empty')
    .trim(),

  tag: z
    .enum(['work', 'ideas', 'personal', 'study'])
    .default('personal'),
})

// TypeScript users: export the inferred type
export type NoteFormData = z.infer<typeof noteSchema>

📄 NoteForm.jsx — RHF + Zod connected

import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { noteSchema } from '../schemas/noteSchema'

function NoteForm({ onSubmit }) {
  const {
    register, // attaches input to RHF
    handleSubmit, // wraps your submit fn
    formState, // contains errors
    reset, // clears the form
  } = useForm({
    resolver: zodResolver(noteSchema),
    defaultValues: { tag: 'personal' },
  })

  const { errors } = formState

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register('title')} />
      {errors.title && <p>{errors.title.message}</p>}
    </form>
  )
}

📄 controllers/noteController.js — GET with optional filter

const Note = require('../models/Note')

const getNotes = async (req, res) => {
  try {
    const { tag } = req.query // ?tag=work
    const filter = tag ? { tag } : {} // if tag exists, filter

    const notes = await Note.find(filter).sort({ createdAt: -1 })
    res.json(notes)
  } catch (err) {
    res.status(500).json({ message: err.message })
  }
}

08 · Build Order

Do it in this exact sequence

Each step compiles in a testable unit. Never skip ahead.

1

Backend: Note model + all 4 routes

Test with Postman or Thunder Client. Make sure GET, POST, PUT, DELETE all work before touching the frontend.

2

Frontend: Static layout, no logic

Build NoteCard and the page layout with hardcoded fake data. Get it looking right first.

3

React Query: fetch and display real notes

Replace your hardcoded data with useQuery. Notes now load from MongoDB.

4

RHF + Zod: the add note form

Write the Zod schema first. Then useForm with zodResolver. Submit → useMutation → invalidate. This is the entire lesson.

5

Edit + Delete actions

Inline edit uses useForm with defaultValues prefilled. Delete uses useMutation same as create.

6

Tag filter (client-side, no API call)

Store the active tag in useState. Filter the notes array locally. Simple, fast — and you'll notice you don't need Zustand for this.
